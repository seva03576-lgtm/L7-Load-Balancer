# L7-Load-Balancer
Высокопроизводительный балансировщик нагрузки уровня L7, написанный на чистом Go. Проект реализует распределение HTTP-трафика без использования сторонних фреймворков.

Архитектура и возможности
Алгоритм Round Robin: Распределение запросов с использованием атомарных операций (sync/atomic), что исключает блокировки (Mutex) на горячем пути.
Active TCP Health Checks: Фоновые горутины каждые 5 секунд проверяют доступность серверов по TCP.
Auto Failover: Упавший сервер моментально исключается из балансировки. Трафик автоматически перенаправляется на живые узлы.
Connection Pooling: Оптимизированное переиспользование TCP-соединений с бэкендами через httputil.ReverseProxy.
Zero Dependencies: Исключительно стандартная библиотека Go.

Нагрузочное тестирование (Стресс-тест)
Тестирование проведено с помощью утилиты hey. Балансировщик обрабатывает более 16 000 RPS (запросов в секунду) на локальной среде без потери пакетов.
```
Команда запуска теста:
```
hey -n 20000 -c 200 http://127.0.0.1:8080
```

Запуск и проверка
Для работы требуются два открытых окна терминала.
```
1. Клонируйте репозиторий:
```bash
git clone https://github.com/seva03576-lgtm/L7-Load-Balancer.git
cd L7-Load-Balancer
```
В первом терминале запустите кластер тестовых бэкендов:
Bash
```
go run backends.go
```
(Поднимутся три сервера на портах 8081, 8082, 8083)

Во втором терминале запустите балансировщик:
Bash
```
go run main.go
```
(Балансировщик запустится на порту 8080)

Откройте браузер по адресу http://127.0.0.1:8080. При обновлении страницы (F5) вы увидите, как запросы циклично распределяются между серверами. При отключении backends.go балансировщик автоматически начнет выдавать ошибку 503 (Service Unavailable), а при частичном отключении — исключит упавшие порты.
